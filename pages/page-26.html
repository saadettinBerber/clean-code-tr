Şirinlik Yapmayın / Don't Be Cute Don't Be Cute
Eğer isimler çok zekiceyse, sadece yazarın mizah anlayışını paylaşanlar tarafından ve sadece mizahı hatırladıkları sürece hatırlanacaktır. HolyHandGrenade (Kutsal El Bombası) isminin bir öğeyi silmek için kullanıldığını kim bilebilir? Belki komiktir, ama belki DeleteItems daha iyidir. Mizah yerine açıklığı (clarity) tercih edin. If names are too clever, they will be memorable only to people who share the author’s sense of humor, and only as long as these people remember the joke. Will they know that the function named HolyHandGrenade is supposed to be the one you call to delete items? Funny, maybe, but maybe DeleteItems is better. Choose clarity over cleverness.
Kodda şirinlik (cuteness) genellikle argo veya deyim kullanımı şeklinde karşımıza çıkar. Örneğin, kill() yerine whack() (tokatlamak/vurmak) veya abort() yerine eatMyShorts() (beni ye/boşver) gibi ifadeler kullanmayın. "Ne demek istiyorsanız onu söyleyin. Ne söylüyorsanız onu kast edin." (Say what you mean. Mean what you say.) Cuteness in code often appears in the form of colloquialisms or slang. For example, don’t use whack() to mean kill(). Don’t tell little culture-dependent jokes like eatMyShorts() to mean abort(). Say what you mean. Mean what you say.
Her Kavram İçin Tek Kelime Seçin / Pick One Word per Concept Pick One Word per Concept
Soyut bir kavram için tek bir kelime seçin ve ona bağlı kalın. Örneğin, farklı sınıflarda eşdeğer metotlar için fetch, retrieve ve get kelimelerinin bir arada kullanılması kafa karıştırıcıdır. Hangi sınıfta hangi ismin kullanıldığını nasıl hatırlayacaksınız? Pick one word for one abstract concept and stick to it. For instance, it’s confusing to have fetch, retrieve, and get as equivalent methods of different classes. How do you remember which method name goes with which class?
Tutarlı bir sözlük (consistent lexicon), o kodu kullanan programcılar için büyük bir nimettir. DeviceManager, ProtocolController ve Driver gibi isimleri aynı projede karmaşık bir şekilde kullanmak yerine, kavramsal olarak benzer yapılar için aynı kelime grubunu (örneğin hep Controller veya hep Manager) tercih etmelisiniz. A consistent lexicon is a great boon to the programmers who must use your code. You might have a DeviceManager, a ProtocolController, and a Driver in the same project. Why not use the same word (e.g., Controller or Manager) for similar concepts?
Kelime Oyunlarından Kaçının / Don't Pun Don't Pun
Aynı kelimeyi iki farklı amaç için kullanmaktan kaçının. Aynı kelimeyi farklı iki fikir için kullanmak, esasen bir kelime oyunudur (pun). Eğer "tek kelime, tek kavram" kuralına uyarsanız, birçok sınıfta örneğin add metodu olabilir. Bu metotlar tutarlı olduğu sürece sorun yoktur (örneğin hepsi yeni bir değer oluşturmak için iki değeri topluyorsa). Avoid using the same word for two different purposes. Using the same word for two different ideas is essentially a pun. If you follow the "one word per concept" rule, you could end up with many classes that have, for example, an add method. As long as those methods are consistent (e.g., they all perform math addition), there's no problem.

Şirinlik Yapmayın (Don't Be Cute)Don't Be Cute

Kavram AçıklamasıConcept Explanation

Kod yazarken zekice veya mizahi isimler kullanmak cazip gelebilir, ancak bu isimler niyetinizi açıkça belirtmez. Profesyonel kod, yazarın mizah anlayışına bağlı kalmadan herkes tarafından anlaşılabilmelidir. Şirinlik ve zeka gösterisi yerine netlik ve sadeliği tercih edin.Using clever or humorous names while coding might be tempting, but these names don't clearly state your intent. Professional code should be understandable by everyone without depending on the author's sense of humor. Choose clarity and simplicity over cuteness and displays of cleverness.

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — Şirinlik ve argo içeren isimlerBAD — Names with cuteness and slang 
// Bir işlemi sonlandırmak için şirinlik yapılmış
public void whack() {
    // ...
}

// Bir hata durumunda durdurmak için deyim kullanılmış
public void eatMyShorts() {
    System.exit(1);
}
whack veya eatMyShorts gibi isimlerin ne işe yaradığını anlamak için kodun içine girmek veya yazara sormak gerekir. Bu, kodun okunabilirliğini düşürür.To understand what names like whack or eatMyShorts do, one would need to dive into the code or ask the author. This reduces code readability.

İyi Örnek (After)Good Example (After)

İYİ — Doğrudan ve profesyonel isimlerGOOD — Direct and professional names 
public void kill() {
    // ...
}

public void abort() {
    System.exit(1);
}
Pratik İpucuPractical Tip"Say what you mean. Mean what you say." (Ne demek istiyorsan onu söyle. Ne söylüyorsan onu kast et.) İlkesini unutmayın. Kodunuz bir edebiyat eseri veya şaka defteri değil, bir iletişim aracıdır.Don't forget the principle: "Say what you mean. Mean what you say." Your code is not a literary work or a joke book; it is a communication tool.

Her Kavram İçin Tek Kelime Seçin (Pick One Word per Concept)Pick One Word per Concept

Kavram AçıklamasıConcept Explanation

Aynı soyut işlem için farklı sınıflarda farklı kelimeler kullanmak (örn: bir yerde fetch, diğerinde get) kafa karıştırıcıdır. Proje genelinde tutarlı bir terminoloji (lexicon) oluşturun. Eğer bir veriyi "getirmek" için get kelimesini seçtiyseniz, her yerde onu kullanın.Using different words for the same abstract operation in different classes (e.g., fetch in one place, get in another) is confusing. Create consistent terminology (lexicon) throughout the project. If you chose the word get to "retrieve" data, use it everywhere.

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — Tutarsız isimlendirmeBAD — Inconsistent naming 
class UserStore {
    public User fetchUser(int id) { ... }
}

class ProductRepository {
    public Product getProduct(int id) { ... }
}

class OrderService {
    public Order retrieveOrder(int id) { ... }
}
Kullanıcı bu metotları çağırmak istediğinde, hangi sınıfın fetch, hangisinin retrieve kullandığını ezberlemek zorunda kalır. Bu durum zihinsel yükü artırır.When the user wants to call these methods, they have to memorize which class uses fetch and which uses retrieve. This increases the mental burden.

İyi Örnek (After)Good Example (After)

İYİ — Tutarlı ve tahmin edilebilir isimlerGOOD — Consistent and predictable names 
class UserStore {
    public User getById(int id) { ... }
}

class ProductRepository {
    public Product getById(int id) { ... }
}

class OrderService {
    public Order getById(int id) { ... }
}
Pratik İpucuPractical TipProjenin başında bir terim sözlüğü oluşturmak faydalıdır. "Veri çekmek için 'get' kullanacağız", "Veriyi kaydetmek için 'save' kullanacağız" gibi kararlar, kodun bütününde tahmin edilebilirliği artırır.It is helpful to create a glossary at the beginning of the project. Decisions like "We will use 'get' to fetch data" or "We will use 'save' to save data" increase predictability throughout the code.

Kelime Oyunlarından Kaçının (Don't Pun)Don't Pun

Kavram AçıklamasıConcept Explanation

Aynı kelimeyi iki farklı kavram için kullanmak bir "kelime oyunu"dur ve okuyucuyu yanıltır. Örneğin, bir listede eleman eklemek için add ismini kullanıyorsanız, ancak başka bir yerde bu kelimeyi "iki sayıyı toplamak" için kullanıyorsanız (anlamsal fark büyükse), bu bir karmaşadır.Using the same word for two different concepts is a "pun" and misleads the reader. For example, if you use the name add to insert an element into a list, but elsewhere you use the same word for "mathematical addition" (if the semantic difference is large), it creates confusion.

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — Aynı kelimenin farklı anlamlarda kullanımıBAD — Using the same word with different meanings 
// Senaryo 1: Koleksiyona eleman eklemek
public void add(Element e) {
    list.add(e);
}

// Senaryo 2: İki değeri toplamak (Pun riski)
public int add(int a, int b) {
    return a + b;
}

// Senaryo 3: Ekrana yazı yazdırmak için "eklemek" (Pun)
public void add(String message) {
    console.println(message);
}
Burada add kelimesi üç farklı amaçla kullanılmıştır. Senaryo 3'te bir şeyleri "bir yere eklemek" değil, aslında "yazdırmak" kastedilmektedir.Here, the word add is used for three different purposes. In Scenario 3, the intent is not to "add something somewhere" but actually to "print" it.

İyi Örnek (After)Good Example (After)

İYİ — Her kavrama özgü net isimlerGOOD — Specific names for each concept 
public void append(Element e) { ... }
public int calculateSum(int a, int b) { ... }
public void displayMessage(String message) { ... }
Pratik İpucuPractical TipBir ismi seçerken "Bu isim başka bir yerde tamamen farklı bir işlem için kullanılıyor mu?" diye sorun. Eğer okuyucu isme bakıp "Buradaki anlamı hangisi?" diye düşünüyorsa, bir kelime oyunu yapmışsınız demektir.When choosing a name, ask "Is this name used elsewhere for a completely different operation?" If the reader looks at the name and wonders "Which meaning is intended here?", you've made a pun.
