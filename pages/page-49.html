Listing 3-7 shows the final refactored version of the messy code (Listing 3-1) we saw at the beginning of this chapter. This class is a concrete example of applying the clean code principles we discussed throughout the chapter. Every function is small, does one thing, and has a meaningful name.
SetupTeardownIncluder.java Listing 3-7 
package fitnesse.html;

import fitnesse.responders.run.SuiteResponder;
import fitnesse.wiki.*;

public class SetupTeardownIncluder {
    private PageData pageData;
    private boolean isSuite;
    private WikiPage testPage;
    private StringBuffer newPageContent;
    private PageCrawler pageCrawler;

    public static String render(PageData pageData) throws Exception {
        return render(pageData, false);
    }

    public static String render(PageData pageData, boolean isSuite)
            throws Exception {
        return new SetupTeardownIncluder(pageData).render(isSuite);
    }

    private SetupTeardownIncluder(PageData pageData) {
        this.pageData = pageData;
        testPage = pageData.getWikiPage();
        pageCrawler = testPage.getPageCrawler();
        newPageContent = new StringBuffer();
    }
Bu kodda dikkat edilmesi gereken birkaç önemli nokta vardır. İlk olarak, sınıfın tüm alanları (fields) private olarak tanımlanmıştır — Kapsülleme (Encapsulation) ilkesine uygun şekilde. İkincisi, dışarıya açılan tek giriş noktası statik render metotlarıdır. Bu, Fabrika Metodu (Factory Method) deseninin basitleştirilmiş bir uygulamasıdır. There are several important points to notice in this code. First, all fields of the class are declared private — following the principle of Encapsulation. Second, the only entry point exposed to the outside world is the static render methods. This is a simplified application of the Factory Method pattern.
Yapıcı metot (constructor) da private olarak tanımlanmıştır, bu sayede sınıf yalnızca kendi statik metotları aracılığıyla oluşturulabilir. Yapıcı, gerekli tüm bağımlılıkları bir kez başlatır ve sınıfın geri kalanını bu başlatılmış değerler üzerinde çalışır hale getirir. The constructor is also declared private, so the class can only be instantiated through its own static methods. The constructor initializes all necessary dependencies once, and the rest of the class operates on these initialized values.

Kapsülleme (Encapsulation)Encapsulation

Kavram AçıklamasıConcept Explanation

Kapsülleme (Encapsulation), bir sınıfın iç durumunu dış dünyadan gizleyerek yalnızca kontrollü bir arayüz (interface) üzerinden erişim sağlama ilkesidir. Alanları private yaparak iç detayları korursunuz.Encapsulation is the principle of hiding a class's internal state from the outside world and providing access only through a controlled interface. By making fields private, you protect internal details.

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — Public alanlarBAD — Public fields 
class UserSession {
    public String token;
    public boolean isExpired;
    public Date lastAccess;
}

// Dışarıdan doğrudan müdahale:
session.isExpired = false; // Tehlikeli!
session.token = "hacked";   // Hiçbir kontrol yok

İyi Örnek (After)Good Example (After)

İYİ — Private alanlar + kontrollü erişimGOOD — Private fields + controlled access 
class UserSession {
    private String token;
    private Instant expiresAt;

    public boolean isExpired() {
        return Instant.now().isAfter(expiresAt);
    }

    public String getToken() {
        if (isExpired())
            throw new SessionExpiredException();
        return token;
    }
}
Pratik İpucuPractical TipVarsayılan olarak her alanı private yapın. Gerçekten dışarıya açılması gerektiğinde bile doğrudan alan yerine bir metot sunun. Bu, ileride iç yapıyı değiştirme özgürlüğü verir.Make every field private by default. Even when you need to expose it, provide a method instead of direct field access. This gives you the freedom to change the internal structure later.

Fabrika Metodu (Factory Method)Factory Method

Kavram AçıklamasıConcept Explanation

Fabrika Metodu (Factory Method), nesne oluşturma mantığını kapsülleyen bir tasarım desenidir. Doğrudan new çağırmak yerine, statik bir metot üzerinden nesne oluşturulur. Bu sayede oluşturma süreci kontrol altına alınır ve iç detaylar gizlenir.The Factory Method is a design pattern that encapsulates object creation logic. Instead of calling new directly, objects are created through a static method. This puts the creation process under control and hides internal details.

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — Doğrudan nesne oluşturmaBAD — Direct object creation 
// İstemci kodu karmaşık kurulumu bilmek zorunda
DatabaseConnection conn = new DatabaseConnection();
conn.setHost("localhost");
conn.setPort(5432);
conn.setCredentials(user, pass);
conn.connect();

İyi Örnek (After)Good Example (After)

İYİ — Statik fabrika metoduGOOD — Static factory method 
class DatabaseConnection {
    private DatabaseConnection() { }

    public static DatabaseConnection create(
            Config config) {
        DatabaseConnection conn =
            new DatabaseConnection();
        conn.initialize(config);
        return conn;
    }
}

// Kullanım: tek satır, temiz
DatabaseConnection conn =
    DatabaseConnection.create(config);
Pratik İpucuPractical TipNesne oluşturmak birden fazla adım gerektiriyorsa veya oluşturma mantığı değişebilirse, statik fabrika metodu kullanın. Yapıcıyı private yaparak istemci kodunun kontrolsüz nesne yaratmasını engelleyin.If creating an object requires multiple steps or the creation logic might change, use a static factory method. Make the constructor private to prevent uncontrolled object creation by client code.

Private Yapıcı Metot (Private Constructor)Private Constructor

Kavram AçıklamasıConcept Explanation

Yapıcı metodu (constructor) private yapmak, sınıfın dışarıdan doğrudan örneklenmesini (instantiation) engeller. Bu teknik genellikle Fabrika Metodu deseniyle birlikte kullanılır ve nesne oluşturma sürecinin tamamen sınıfın kontrolünde kalmasını sağlar.Making a constructor private prevents the class from being directly instantiated from outside. This technique is commonly used with the Factory Method pattern to ensure that the object creation process remains entirely under the class's control.

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — Public yapıcı, kontrolsüz oluşturmaBAD — Public constructor, uncontrolled creation 
class Logger:
    def __init__(self, file_path):
        self.file = open(file_path, "a")

# Her yerde yeni logger oluşturulabilir
logger1 = Logger("/var/log/app.log")
logger2 = Logger("/var/log/app.log")
# Aynı dosyaya iki farklı handler — sorun!

İyi Örnek (After)Good Example (After)

İYİ — Kontrollü oluşturmaGOOD — Controlled creation 
class Logger:
    _instance = None

    def __init__(self):
        raise RuntimeError("Use Logger.get()")

    @classmethod
    def get(cls, file_path):
        if cls._instance is None:
            obj = object.__new__(cls)
            obj.file = open(file_path, "a")
            cls._instance = obj
        return cls._instance

# Her zaman aynı nesne döner
logger = Logger.get("/var/log/app.log")
Pratik İpucuPractical TipYapıcıyı private yapmak "Bu nesneyi nasıl oluşturacağıma ben karar veririm" demektir. SetupTeardownIncluder'da olduğu gibi, kullanıcının render() çağırması yeterli — sınıfın iç mekanizmasını bilmesine gerek yoktur.Making the constructor private means "I decide how this object is created." As in SetupTeardownIncluder, the user just calls render() — they don't need to know the class's internal mechanics.
