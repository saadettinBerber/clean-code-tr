Sık Görülen Tekli Formlar / Common Monadic Forms Common Monadic Forms
Bir argümanı bir fonksiyona geçirmek için iki çok yaygın neden vardır. Argüman hakkında bir soru soruyor olabilirsiniz, örneğin: boolean fileExists(String fileName). Ya da argüman üzerinde bir işlem yapıyor, onu başka bir şeye dönüştürüp geri döndürüyor olabilirsiniz, örneğin: InputStream fileOpen(String fileName). There are two very common reasons to pass a single argument into a function. You may be asking a question about that argument, as in boolean fileExists(String fileName). Or you may be operating on that argument, transforming it into something else and returning it, as in InputStream fileOpen(String fileName).
Bu iki kullanım dışında, tek argümanlı bir fonksiyon kullanmaya çalışmak dikkatli düşünmeyi gerektirir. Örneğin, bir void dönen ve tek argüman alan bir fonksiyon, o argüman üzerinde yan etki (side effect) yaratıyor olabilir. Bu durumda fonksiyon ismi bu yan etkiyi açıkça belirtmelidir. Now try to use a single-argument function outside of these two cases and you should think carefully. For example, a function that returns void and takes a single argument might be creating a side effect on that argument. In this case, the function name should clearly indicate this side effect.
Bayrak Argümanları / Flag Arguments Flag Arguments
Bayrak argümanları (flag arguments) çirkindir. Bir fonksiyona boolean değer geçirmek, fonksiyonun birden fazla şey yaptığını yüksek sesle ilan eder. Doğruysa bir şey yapar, yanlışsa başka bir şey yapar! Flag arguments are ugly. Passing a boolean into a function loudly proclaims that this function does more than one thing. It does one thing if the flag is true and another if the flag is false!
Liste 3-7'deki render(boolean isSuite) fonksiyonunu ele alalım. Bu fonksiyon, isSuite değeri true olduğunda farklı, false olduğunda farklı bir şey yapar. Bu fonksiyon iki fonksiyona bölünmelidir: renderForSuite() ve renderForSingleTest(). Consider the render(boolean isSuite) function in Listing 3-7. This function does different things depending on whether the isSuite value is true or false. This function should be split into two functions: renderForSuite() and renderForSingleTest().
Evet, bu iki fonksiyonun bazı kodları tekrar etmesi gerekebilir. Ancak bu tekrar, daha iyi okunabilirlik ve daha temiz bir API'nin bedelidir. Bayrak argümanlarından kaçının! Yes, these two functions might duplicate some code. But this duplication is the price of better readability and a cleaner API. Avoid flag arguments!
İkililer / Dyadic Functions Dyadic Functions
İki argümanlı bir fonksiyon, tek argümanlı bir fonksiyondan daha zor anlaşılır. İki girdiyi anlamak için kısa bir duraksama vardır. Örneğin: InputStream fileOpen(String fileName, String mode) vs InputStream fileOpen(String fileName). A function with two arguments is harder to understand than a monadic function. There is a short pause in understanding the two inputs. For example: InputStream fileOpen(String fileName, String mode) vs InputStream fileOpen(String fileName).
İkililer kullanmak kötü bir uygulama değildir, ancak maliyetinin farkında olmalı ve bunları ikiliye dönüştürmenin bir yolu olup olmadığını sormalısınız. Örneğin, writeField(String name, String value) metodu, writeField(Field field) şeklinde yazılarak tekli hale getirilebilir. Using dyads is not an evil practice, but you should be aware of the cost and ask whether there is a way to convert them into monads. For example, the writeField(String name, String value) method can be made monadic by writing it as writeField(Field field).

Tekli Formlar (Monadic Forms)Monadic Forms

Kavram AçıklamasıConcept Explanation

Tek argümanlı fonksiyonların iki yaygın formu vardır: (1) Argüman hakkında soru soran (boolean fileExists), (2) Argümanı dönüştürüp yeni değer döndüren (InputStream fileOpen).Single-argument functions have two common forms: (1) Asking a question about the argument (boolean fileExists), (2) Transforming the argument and returning a new value (InputStream fileOpen).

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — Void + tek argüman (yan etki)BAD — Void + single argument (side effect) 
void process(String data) {
    // Ne yapıyor? Belirsiz!
    saveToFile(data);
}

İyi Örnek (After)Good Example (After)

İYİ — Açık isim veya soruGOOD — Clear name or question 
boolean isValidEmail(String email);
Report generateReportFrom(Data data);
void saveToDatabase(User user); // Void ise yan etki açık
Pratik İpucuPractical TipTek argümanlı void fonksiyonlardan kaçının. Ya bir değer döndürün (dönüşüm) ya da isim yan etkiyi açıkça belirtsin.Avoid single-argument void functions. Either return a value (transformation) or make the name clearly indicate the side effect.

Bayrak Argümanları (Flag Arguments)Flag Arguments

Kavram AçıklamasıConcept Explanation

Boolean argümanlar fonksiyonun birden fazla şey yaptığını gösterir. Doğruysa bir şey, yanlışsa başka bir şey yapar. Bu iki ayrı fonksiyon olmalıdır.Boolean arguments indicate a function does more than one thing. Does one thing if true, another if false. These should be two separate functions.

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — Bayrak argümanıBAD — Flag argument 
def render(data, isSuite):
    if isSuite:
        renderSuite(data)
    else:
        renderSingle(data)

# Kullanım:
render(data, True)  # True ne anlama geliyor?

İyi Örnek (After)Good Example (After)

İYİ — Ayrı fonksiyonlarGOOD — Separate functions 
def renderForSuite(data):
    renderSuite(data)

def renderForSingleTest(data):
    renderSingle(data)

# Kullanım:
renderForSuite(data)      # Açık!
renderForSingleTest(data) # Açık!
Pratik İpucuPractical TipBoolean parametre gördüğünüzde alarm çalması gerekir. "Bu fonksiyon iki şey mi yapıyor?" diye sorun. Eğer öyleyse, ikiye bölün.When you see a boolean parameter, alarm bells should ring. Ask: "Is this function doing two things?" If yes, split it in two.

İkili Fonksiyonlar (Dyadic Functions)Dyadic Functions

Kavram AçıklamasıConcept Explanation

İki argümanlı fonksiyonlar tekliye göre daha zor anlaşılır. Argüman sırası hatırlanmalıdır. Mümkünse ikilileri tekliye dönüştürün.Two-argument functions are harder to understand than monads. Argument order must be remembered. Convert dyads to monads when possible.

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — Ayrı argümanlarBAD — Separate arguments 
def writeField(name, value):
    # Hangisi önce? Hangisi sonra?
    ...

writeField("email", "user@test.com")
writeField("user@test.com", "email")  # Hata!

İyi Örnek (After)Good Example (After)

İYİ — Nesne olarak gruplamaGOOD — Grouped as object 
class Field:
    def __init__(self, name, value):
        self.name = name
        self.value = value

def writeField(field):
    # Tek argüman - nesne
    ...

writeField(Field("email", "user@test.com"))
Pratik İpucuPractical TipArgümanlar birlikte geziyorsa (her zaman birlikte kullanılıyorsa), bir nesne yapın. "Bu iki argüman birbirine ait mi?" sorusunu sorun.When arguments travel together (always used together), make them an object. Ask: "Do these two arguments belong together?"
