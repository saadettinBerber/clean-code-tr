Bloklar ve Girinti / Blocks and Indenting Blocks and Indenting
Bu, if ifadeleri, else ifadeleri, while ifadeleri ve benzerleri içindeki blokların bir satır uzunluğunda olması gerektiği anlamına gelir. Muhtemelen bu satır bir fonksiyon çağrısı olmalıdır. Bu, sadece fonksiyonu küçük tutmakla kalmaz, ayrıca blok içinde çağrılan fonksiyonun güzel bir açıklayıcı isme sahip olması nedeniyle belgesel değer de katar. This implies that the blocks within if statements, else statements, while statements, and so on should be one line long. Probably that line should be a function call. Not only does this keep the enclosing function small, but it also adds documentary value because the function called within the block can have a nicely descriptive name.
Bu aynı zamanda fonksiyonların iç içe yapıları (nested structures) tutacak kadar büyük olmaması gerektiği anlamına da gelir. Bu nedenle, bir fonksiyonun girinti seviyesi (indent level) bir veya ikiyi geçmemelidir. Bu, elbette fonksiyonların okunmasını ve anlaşılmasını kolaylaştırır. This also implies that functions should not be large enough to hold nested structures. Therefore, the indent level of a function should not be greater than one or two. This, of course, makes the functions easier to read and understand.

Bloklar ve Girinti (Blocks and Indenting)Blocks and Indenting

Kavram AçıklamasıConcept Explanation

if, while, for gibi bloklar bir satır olmalıdır - bu satır genellikle bir fonksiyon çağrısıdır. Bu kural, fonksiyonların küçük kalmasını sağlar ve okunabilirliği artırır. Ayrıca fonksiyonun girinti seviyesi 1-2'yi geçmemelidir.if, while, for blocks should be one line - usually a function call. This rule keeps functions small and improves readability. Also, function indent level should not exceed 1-2.

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — Çok satırlı bloklarBAD — Multi-line blocks 
if (user.isActive()) {
    List permissions = ArrayList();
    for (Role role : user.getRoles()) {
        if (role.hasPermission("READ")) {
            permissions.add(role.getName());
        }
    }
    return permissions;
}

İyi Örnek (After)Good Example (After)

İYİ — Tek satırlı bloklarGOOD — Single-line blocks 
if (user.isActive())
    return extractReadPermissions(user.getRoles());

private List extractReadPermissions(List<Role> roles) {
    return roles.stream()
        .filter(r -> r.hasPermission("READ"))
        .map(r -> r.getName())
        .collect(toList());
}
Pratik İpucuPractical TipBlok içinde birden fazla satır varsa, o kodu hemen bir fonksiyona çıkarın. İsimlendirmesi de önemlidir - ne yaptığını anlatan açıklayıcı bir isim seçin.If a block has multiple lines, extract that code into a function immediately. Naming is also important - choose a descriptive name that explains what it does.

İç İçe Yapılar (Nested Structures)Nested Structures

Kavram AçıklamasıConcept Explanation

Fonksiyonlardaki aşırı iç içe geçme (over-nesting), kodun okunmasını ve anlaşılmasını zorlaştırır. Her seviye girinti, okuyucunun zihinsel yükünü artırır. Martin, fonksiyon girinti seviyesinin 1-2'yi geçmemesi gerektiğini söyler. Daha derin iç içe geçmeler, fonksiyonun bölünmesi gerektiğinin işaretidir.Excessive nesting in functions makes code hard to read and understand. Each indent level increases the reader's mental load. Martin says function indent level should not exceed 1-2. Deeper nesting is a sign the function should be split.

Kötü Örnek (Before)Bad Example (Before)

KÖTÜ — 4 seviye iç içe geçmeBAD — 4 levels deep 
function processData(data) {
    if (data) {
        if (data.items) {
            for (let item of data.items) {
                if (item.isValid) {
                    save(item);
                }
            }
        }
    }
}

İyi Örnek (After)Good Example (After)

İYİ — Düz, okunabilir yapıGOOD — Flat, readable structure 
function processData(data) {
    if (!data?.items) return;
    
    const validItems = filterValidItems(data.items);
    validItems.forEach(save);
}

function filterValidItems(items) {
    return items.filter(i => i.isValid);
}
Pratik İpucuPractical Tip"Ok koridoru" (corridor of doom) oluşmaya başladığında - yani sürekli sağa kaydığınızda - bu, fonksiyonları bölme zamanı gelmiştir.When you start creating a "corridor of doom" - constantly shifting right - it's time to split the functions.
