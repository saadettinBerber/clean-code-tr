Temiz Kod (Clean Code) Clean Code

Kavram Aciklamasi Concept Explanation

Temiz Kod (Clean Code), okunabilir, anlasilabilir ve bakimi kolay kod yazma disiplinidir. Degisken, fonksiyon ve sinif isimlendirmesi bu disiplinin en temel yapitasidir. Iyi isimlendirilmis kod, yorum satirlarina ihtiyac duymadan kendini aciklar. Clean Code is the discipline of writing readable, understandable, and maintainable code. Naming of variables, functions, and classes is the most fundamental building block of this discipline. Well-named code explains itself without needing comments.

Kotu Ornek (Before) Bad Example (Before)

KOTU BAD 
def p(l):
    t = 0
    for i in l:
        if i > 0:
            t = t + i
    return t
Fonksiyon ve degisken isimleri tamamen anlamsiz. p, l, t, i gibi tek harfli isimler kodun ne yaptigini gizliyor. Function and variable names are completely meaningless. Single-letter names like p, l, t, i hide what the code does.

Iyi Ornek (After) Good Example (After)

IYI GOOD 
def sum_positive_numbers(numbers):
    total = 0
    for number in numbers:
        if number > 0:
            total += number
    return total
Fonksiyon adi (sum_positive_numbers) tam olarak ne yaptigini soyluyor. Degisken adlari (numbers, total, number) baglami acikca ortaya koyuyor. The function name (sum_positive_numbers) tells exactly what it does. Variable names (numbers, total, number) clearly convey context.
Pratik Ipucu Practical Tip Bir fonksiyon veya degisken adlandirirken kendinize sunu sorun: "Bu kodu 6 ay sonra okudugumda ne yaptigini anlayabilir miyim?" Eger cevap hayirsa, ismi degistirin. When naming a function or variable, ask yourself: "Will I understand what this does 6 months from now?" If the answer is no, rename it.

Yazilim Zanaati (Software Craftsmanship) Software Craftsmanship

Kavram Aciklamasi Concept Explanation

Yazilim Zanaati (Software Craftsmanship), yazilim gelistirmeyi sadece bir muhendislik dalasi degil, ayni zamanda bir zanaat olarak goren yaklasimdir. "Calisiyor" demek yeterli degildir; kodun ayni zamanda ozenle, dikkatle ve guzellikle yazilmis olmasi gerekir. Bu, profesyonellik ve sorumluluk bilinciyle kod yazmak demektir. Software Craftsmanship is an approach that views software development not just as an engineering discipline, but also as a craft. "It works" is not enough; code must also be written with care, attention, and elegance. This means writing code with professionalism and a sense of responsibility.

Kotu Ornek (Before) Bad Example (Before)

KOTU BAD 
// TODO: fix this later
// HACK: don't know why this works
function calcPrice(x, y, z) {
    var r = x * y;
    if (z) r = r - (r * 0.1); // 10% discount maybe?
    return r;
}
Bu kod "isini olsun bitsin" zihniyetiyle yazilmis. Gecici cozumler (hack), belirsiz TODO'lar, anlamsiz degisken adlari ve sihirli sayilar (magic numbers) iceriyor. This code was written with a "just get it done" mentality. It contains hacks, vague TODOs, meaningless variable names, and magic numbers.

Iyi Ornek (After) Good Example (After)

IYI GOOD 
const LOYALTY_DISCOUNT_RATE = 0.10;

function calculateTotalPrice(unitPrice, quantity, isLoyalCustomer) {
    const subtotal = unitPrice * quantity;

    if (isLoyalCustomer) {
        return subtotal - (subtotal * LOYALTY_DISCOUNT_RATE);
    }

    return subtotal;
}
Bu kod ozenle yazilmis. Sabitler anlamli isimlerle tanimlanmis (LOYALTY_DISCOUNT_RATE), fonksiyon ve parametre adlari kendi kendini acikliyor. This code was written with care. Constants have meaningful names (LOYALTY_DISCOUNT_RATE), function and parameter names are self-documenting.
Pratik Ipucu Practical Tip Her commit yapmadan once kodunuza bir adim geriden bakin. "Bu kodu bir is arkadasima gostermekten gurur duyar miyim?" diye sorun. Before every commit, step back and look at your code. Ask yourself: "Would I be proud to show this to a colleague?"
